# Incident Knowledge Dataset - 50 Real-World IT Incidents

## Database Incidents

1. **PostgreSQL Connection Pool Exhausted**
   Category: DATABASE_CONNECTION_ERROR
   Severity: HIGH
   Symptoms: Application throwing "connection pool exhausted" errors, users unable to login, database connections at maximum limit
   Root Cause: High traffic spike combined with long-running queries holding connections, inadequate connection pool sizing
   Solution: 1. Kill long-running queries 2. Increase connection pool size 3. Implement connection timeout 4. Add connection monitoring 5. Optimize slow queries
   Tags: postgresql, connection-pool, performance, database

2. **MySQL Deadlock Storm**
   Category: DATABASE_DEADLOCK
   Severity: MEDIUM
   Symptoms: Multiple transactions failing with deadlock errors, application retries failing, database performance degraded
   Root Cause: Concurrent transactions accessing same tables in different order, inadequate indexing causing lock escalation
   Solution: 1. Identify deadlock patterns from logs 2. Reorder transaction operations 3. Add proper indexes 4. Implement retry logic with exponential backoff 5. Consider row-level locking
   Tags: mysql, deadlock, transactions, concurrency

3. **MongoDB Replica Set Primary Down**
   Category: DATABASE_OUTAGE
   Severity: CRITICAL
   Symptoms: Write operations failing, application in read-only mode, replica set election in progress
   Root Cause: Primary node hardware failure, network partition causing split-brain scenario
   Solution: 1. Check replica set status 2. Force election if needed 3. Investigate primary node failure 4. Update connection strings if needed 5. Monitor replication lag
   Tags: mongodb, replica-set, failover, high-availability

4. **Redis Memory Limit Reached**
   Category: CACHE_MEMORY_FULL
   Severity: HIGH
   Symptoms: Cache writes failing, application performance degraded, Redis evicting keys unexpectedly
   Root Cause: Memory usage exceeded maxmemory setting, inadequate eviction policy, memory leaks in cached data
   Solution: 1. Increase Redis memory limit 2. Configure appropriate eviction policy 3. Clear unnecessary keys 4. Implement TTL for cached data 5. Monitor memory usage
   Tags: redis, memory, cache, eviction-policy

5. **Oracle Tablespace Full**
   Category: DATABASE_STORAGE_FULL
   Severity: HIGH
   Symptoms: INSERT operations failing with "tablespace full" error, application unable to save data
   Root Cause: Rapid data growth, inadequate storage monitoring, missing tablespace auto-extend configuration
   Solution: 1. Add datafiles to tablespace 2. Enable auto-extend 3. Archive old data 4. Implement storage monitoring 5. Plan capacity expansion
   Tags: oracle, tablespace, storage, capacity-planning
